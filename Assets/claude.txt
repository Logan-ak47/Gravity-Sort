● Done. A few design notes worth calling out:

  SetSelected(bool selected) — always kills tweens and resets scale to
  Vector3.one first, regardless of the selected value. This prevents scale bleed
   if SetSelected(true) is called while a pulse is already running.

  PlayClearAnimation — splits clearDuration in half: first half scales to 1.2
  (OutQuad), second half fades alpha to 0 (OutQuad), matching the GDD's "Scale
  1.0→1.2 → fade out" spec. Uses a DOTween Sequence so the two tweens chain
  correctly before firing onComplete.

  PlayPourArc — uses DOJump with numJumps: 1 which gives a clean single arc.
  arcHeight comes from GameConfig.pourArcHeight (1.5f default). The caller
  passes it in so the arc height can be adjusted per-pour if needed.

  DOTween.Kill(transform) — targets tweens by the transform, which covers all
  tweens started via transform.DO* or spriteRenderer.DO* that were chained off
  the same target. For the spriteRenderer fade specifically, note that DOTween
  registers the SpriteRenderer as the tween target — if you ever need to kill
  those independently, use DOTween.Kill(spriteRenderer). For now the Sequence
  approach handles it cleanly since the kill fires before the sequence starts.
